## 📋 인수인계 문서 개요

본 인수인계 문서는 AI CS 시스템의 전체 구성, 개발 내역, 운영 방법을 포함한 종합 기술 문서입니다.
개발자가 시스템을 이해하고 유지보수할 수 있도록 필요한 모든 정보를 제공합니다.

### 3. API 명세서
**파일명**: `03_API_명세서.md`

**목적**: 백엔드 API의 상세한 명세 제공

**주요 내용**:
- API 개요 및 인증 방식
- 인증 API (로그인, 사용자 정보 조회)
- 대시보드 API (통계 및 차트 데이터)
- 이벤트 API (검색, 상세 조회, 상담 문구 생성)
- 관리자 API (플랫폼/브랜드 관리)
- 즐겨찾기 API
- 에러 코드 정의
- 공통 사항 (페이지네이션, 정렬, 필터링)

**대상 독자**: 프론트엔드 개발자, 백엔드 개발자, API 사용자

**페이지 수**: 약 40페이지

**주요 API 엔드포인트**:
- `POST /api/auth/login` - 로그인
- `GET /api/dashboard` - 대시보드 데이터
- `GET /api/events/search` - 이벤트 검색
- `GET /api/events/:id` - 이벤트 상세
- `GET /api/admin/platforms` - 플랫폼 목록
- `POST /api/favorites` - 즐겨찾기 추가

---

### 4. 데이터베이스 설계서
**파일명**: `04_데이터베이스_설계서.md`

**목적**: 데이터베이스 스키마 및 설계 원칙 이해

**주요 내용**:
- 데이터베이스 개요 (PostgreSQL 15, Supabase)
- ERD (Entity Relationship Diagram)
- 테이블 명세 (15개 테이블)
  - channels, platforms, brands
  - live_broadcasts, live_products, live_coupons
  - live_benefits, live_comments, live_faqs
  - live_intro, live_statistics, live_images
  - cs_response_scripts, expected_customer_questions, risk_points
- 인덱스 설계
- 제약 조건 (PK, FK, Unique, Check)
- 데이터 타입 및 명명 규칙
- 백업 및 복구 절차

**대상 독자**: 데이터베이스 관리자, 백엔드 개발자

**페이지 수**: 약 50페이지

**주요 테이블**:
- `live_broadcasts`: 라이브 방송 기본 정보
- `live_products`: 상품 정보 (~18,000/월)
- `live_coupons`: 쿠폰 정보
- `cs_response_scripts`: CS 응답 스크립트
